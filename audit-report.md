# Аудит та виправлення банку питань CCA-тренажера

**Дата:** 2026-06-04 · **Банк:** 125 питань · `lib/exam-bank.ts`

## Резюме

Виявлено й усунуто два системні дефекти банку:

| Метрика | До | Після | Ціль |
|---|---|---|---|
| `ex` з посиланнями на літери варіантів | **103 / 125 (82%)** | **0 / 125 (0%)** | 0% |
| Правильна відповідь = найдовша (унікально) | **118 / 125 (94%)** | **26 / 125 (21%)** | ≤30% |
| Правильна відповідь = найкоротша | — | 25 / 125 (20%) | ~рівномірно |
| Правильна >1.5× за сер. дистрактор | 98 | 0 | — |
| Сер. відношення max/min довжини опцій | ~1.9 | 1.11 | →1.0 |
| Структурні помилки / дублікати id | 0 | 0 | 0 |

Розподіл рангу довжини правильної відповіді тепер ~рівномірний (21% найдовша / 20% найкоротша / 59% посередині-нічия) — довжина більше **не є підказкою** в жоден бік.

## Що зроблено (Задачі 1–3, одним наскрізним проходом)

Для кожного з 125 питань:

1. **Прибрано всі літерні посилання з `ex`** — «(A)», «(B)», «(B, C)», кириличні «(А)»…«(Г)», «варіант X», «відповідь X». Замість літери варіант описується за змістом/концепцією. Пояснення тепер коректні **незалежно від перемішування** варіантів (тренажер перемішує опції при показі — це коректна механіка, її не чіпали).

2. **Аудит правильності** за 4 пунктами (правильна відповідь дійсно правильна; `ex` відповідає їй і фактично коректне; рівно 4 варіанти з одним однозначно правильним; коректна українська й термінологія). Звірка з офіційним Exam Guide і best practices Anthropic (agentic loops/stop_reason, hooks, coordinator-subagent, MCP tool design, structured errors, tool_choice, Claude Code, few-shot, tool_use JSON-схеми, nullable, Batches API, context management).

3. **Вирівняно довжину варіантів** — зайву деталізацію з правильного варіанта перенесено в `ex`; дистрактори дописано до співмірної довжини й конкретності (правдоподібні, але технічно хибні). Суть і правильність збережено: один правильний, три хибні, той самий індекс `a`.

**Процес:** мультиагентний воркфлоу — рерайт-агент + незалежний верифікатор на кожен пакет питань; далі два цільові проходи на вирівнювання довжини й балансування рангу. Кожне виправлення перевірено окремим агентом-рев'юером; фінал звірено детермінованим лінтом (`scripts/audit-bank.ts`).

## Спірні випадки — винесено на рішення власника (НЕ змінювалися мовчки)

### S2Q5 — модульність CLAUDE.md
Правильна відповідь (декомпозиція великого CLAUDE.md на тематичні файли + `@import`) — **коректна**. Агент засумнівався у згадці теки `.claude/rules/` як офіційної конвенції.
**Резолюція:** офіційний Exam Guide прямо описує `.claude/rules/` (Task Statement 3.3 «path-specific rules», `.claude/rules/` з YAML frontmatter). Деталь підтверджена, відповідь коректна. **Змін не потрібно.**

### S5Q4 — машиночитний вивід у CI
Правильна відповідь (`--output-format json` + `--json-schema`) — **коректна**. Агент засумнівався у точній назві прапорця `--json-schema`.
**Резолюція:** Exam Guide прямо перелічує «`--output-format json`, `--json-schema` for structured CI output» (секція Claude Code CLI; Task 3.6). Прапорець підтверджений. **Змін не потрібно.**
